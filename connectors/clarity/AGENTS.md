# Microsoft Clarity Connector

Behavior and friction signals (dead clicks, rage clicks, scroll depth, script errors) via the Clarity Data Export API.

## Status

Active. Clarity is free and unlimited; the Data Export API needs a token but costs nothing. This is the qualitative "why" layer that pairs with GA4's quantitative "what" under the Conversion Expert.

## What This Connector Is (and Is Not)

The Data Export API returns aggregated dashboard insights only. It does NOT export session recordings or heatmap images; those stay in the Clarity UI for a human to watch. This connector pulls the structured signals that tell you which pages and sessions are worth opening.

## Quick Start

```bash
node scripts/insights.js signals --days 3
```

Needs a token. If you get "Cannot find module", run `npm install` first.

## Configuration

**Credentials (required):** `/memory/connectors/clarity/.env`

`/memory/` is a workspace root. Resolve from the workspace paths before reading or creating this file.

```
CLARITY_API_TOKEN=your_jwt_token
```

The token is a JWT generated by a project admin at Settings > Data Export > Generate new API token. It is scoped to one Clarity project (one site); set up one `.env` per site, or swap the token when auditing a different site. Walk the user through `SETUP.md`.

## Scripts

| Script | Commands | Purpose |
|--------|----------|---------|
| `auth.js` | `check` | Confirm the token is valid (uses 1 of 10 daily requests) |
| `insights.js` | `insights`, `signals` | Raw Live Insights, or a friction-focused digest |

Run any script with `help` for full syntax.

## Key Commands

| Command | What It Returns |
|---------|----------------|
| `node scripts/insights.js signals --days 3` | Friction digest (dead/rage/error clicks, quick backs, excessive scroll, script errors) + traffic, site-wide |
| `node scripts/insights.js signals --days 3 --by URL` | Same digest broken down per page (find which pages cause friction) |
| `node scripts/insights.js insights --days 1 --dimensions URL,Device` | Raw Live Insights broken down by up to 3 dimensions |
| `node scripts/auth.js check` | Validate the token |

**Dimensions** (max 3 per request): Browser, Device, Country, OS, Source, Medium, Campaign, Channel, URL.

**Friction metrics** returned: Dead Click Count, Rage Click Count, Quickback Click, Excessive Scroll, Script Error Count, Error Click Count. Plus Traffic, Scroll Depth, and Engagement Time.

## Quota and Window (read before pulling)

The Data Export API is tightly limited. Plan calls deliberately:

- **10 requests per project per day.** Each `insights`/`signals`/`auth check` call counts. The Conversion Expert should batch a run into a few targeted calls, not loop.
- **Last 1 to 3 days only.** There is no historical range. For trends, pull on a cadence and store snapshots yourself.
- **Max 3 dimensions per request**, response capped at 1,000 rows, no pagination.
- Results are in UTC.

## Relationship to GA4 and the Conversion Expert

GA4 (via the `google` connector, `analytics.js`) answers WHERE users drop off (pages, funnels, sources). This connector answers WHY (rage clicks on a dead element, a form field that triggers quick backs, a script error on checkout). The Conversion Expert reads both: GA4 to find the leak, Clarity to explain it, then prioritizes the fix. Replays and heatmaps for the flagged pages are reviewed by a human in the Clarity UI.

## Troubleshooting

**"CLARITY_API_TOKEN not found":** Create `/memory/connectors/clarity/.env`. Follow `SETUP.md`.

**401 Unauthorized:** Token missing, invalid, or expired. Regenerate at Settings > Data Export. Replace the token in `.env`.

**403 Forbidden:** Token is not authorized for that project. Confirm you generated it inside the correct Clarity project.

**429 Too Many Requests:** Daily limit reached (10/day). Wait until the next UTC day, or reduce call count.

**Empty `information` arrays:** The site had no traffic in the window, or Clarity is not installed/collecting on the site yet. Confirm the Clarity tag is live on the site.
