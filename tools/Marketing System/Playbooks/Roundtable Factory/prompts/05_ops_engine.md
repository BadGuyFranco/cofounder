# 05 — Operations Engine

## Purpose
Ensure the roundtable runs smoothly and improves over time. Consolidates Ops Checklist, Tech Setup, and QA/Scorecard.

**Goal**: Produce `05_Operations_Manual.md`.

---

## Components

### 1. Logistics
- **Tech Stack**: Zoom settings, Calendar links, Recording tools.
- **Timeline**: When to open the room, when to hit record.

### 2. Scaling
- **Delegation**: What a VA can handle (invites, chat monitoring).
- **Frequency**: How often to run this.

### 3. Quality Assurance (Scorecard)
- **Self-Audit**: Did I talk too much? Did I mine deep enough?
- **Metrics**: Show rate, Book rate, Close rate.

---

## Output Structure: `05_Operations_Manual.md`

```markdown
# Operations Manual

## 1. Pre-Flight Checklist
- [ ] Zoom link tested
- [ ] Slides ready
- [ ] LinkedIn invites sent

## 2. In-Flight Protocol
- [ ] Record to cloud
- [ ] Paste welcome chat

## 3. Post-Flight Protocol
- [ ] Send replay to "No Shows"
- [ ] Update CRM

## 4. The Scorecard
- [ ] Did I stick to 60 mins?
- [ ] Did 20%+ book a call?
```
